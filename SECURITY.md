# Security Policy

## Supported Scope
- Latest `main` branch
- Latest tagged release artifacts

## Reporting a Vulnerability
1. Do not open public issues for unpatched vulnerabilities.
2. Report details privately to project maintainers through your hosting platform's private security reporting channel.
3. Include:
   - affected component/file
   - impact and attack path
   - reproduction steps
   - suggested fix (if available)

## Handling Guidelines
- Acknowledge report receipt promptly.
- Validate and triage severity.
- Patch with tests and release notes.
- Disclose publicly after fix availability.
